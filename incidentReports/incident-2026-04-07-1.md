# Incident: 2026-04-09 20-13-00

## Summary

> [!NOTE]
> Write a summary of the incident in a few sentences. Include what happened, why, the severity of the incident and how long the impact lasted.

```md
My second chaos attempt started between 12pm to 2pm on 2026-04-09, users trying to complete a order pizza on checkout could not place orders on the website. The UI showed **Failed to fulfill order at factory**, and the network browser tab showed `POST https://pizza-service.cs329pizzawebsite.click/api/order` returning **500 Internal Server Error**. The JSON error body included **`followLinkToEndChaos`**, pointing to a JWT Pizza Factory support/report URL. The vendor dashboard showed **Chaos status: chaotic** during the outage. The event was triggered by **planned vendor-side chaos** for CS 329 Deliverable 11. The pizza service behaved as designed that was when the factory returned a non-success response, the API responded with **Failed to fulfill order at factory** and the factory-provided remediation link. This was a **major** functional incident for the ordering path for the duration of active factory chaos, but impact ended within roughly **tens of minutes** once the support URL was used and chaos returned to **calm**.
```

## Root cause

> [!NOTE]
> Note the final root cause of the incident, the thing identified that needs to change in order to prevent this class of incident from happening again.

```md
**Root cause:** Intentional **JWT Pizza Factory vendor chaos** for Deliverable 11. While chaos was active, the factory did not successfully fulfill orders for the vendor; the pizza service propagated that as **Failed to fulfill order at factory** and returned the factory **`reportUrl`** as `followLinkToEndChaos`.
**What must change operationally (not in code):** Chaos should only run in controlled windows with alerting and traffic simulation, and must be cleared via the factory report flow or vendor dashboard so production-like ordering can resume.
```
